# Product Specs Index

Use `docs/product-specs/` for statements of product intent: audience, goals, scope, constraints, and acceptance criteria.

## Organization

- One spec per feature, landing variant, or meaningful behavior change
- Prefer filenames like `YYYY-MM-DD-short-title.md`
- Link the matching execution plan and any design doc that implements the spec

## Include

- problem or opportunity
- target user or audience
- scope and non-goals
- acceptance criteria
- launch or rollout notes when relevant

Keep specs concise and decision-oriented.
