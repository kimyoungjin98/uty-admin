# Design Docs Index

Use `docs/design-docs/` for documents about interaction design, content structure, visual systems, and notable UX decisions.

## Organization

- One document per feature, redesign, or significant UX decision
- Prefer filenames like `YYYY-MM-DD-short-title.md`
- Link related execution plans, product specs, and screenshots or references

## Include

- problem statement
- decision summary
- key states and flows
- affected routes or components
- open questions and follow-ups

Skip design docs for trivial copy changes or isolated spacing tweaks.
