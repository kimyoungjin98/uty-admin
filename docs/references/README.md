# References

Use this folder for long-lived reference material that helps future work but is not itself a plan, spec, or design decision.

Good candidates:

- vendor integration notes
- deployment checklists
- browser or platform constraints
- architecture diagrams
- curated external links with short summaries

Rules:

- Prefer durable material over temporary debugging notes.
- Add source links and dates for external references.
- Summarize why the reference matters to this repo.
- Move active decision-making into plans, specs, or design docs instead of storing it here.
